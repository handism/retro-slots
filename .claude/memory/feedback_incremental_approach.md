---
name: 大きな変更は小さく分割して段階的に実装する
description: 複雑な実装は小さなテスタブルな単位に分割し、各ステップで検証する
type: feedback
---

複雑な実装や複数ファイルにまたがる変更は、一度に大量のコードを生成せず、小さな単位に分割して段階的に進める。

**ルール:**
1. 大きな変更は「最小単位のステップ」に分割してから着手する
2. 各ステップ完了後に検証が必要な場合はユーザーに確認を求めてから次のステップへ進む
3. PLAN.md の各チェックボックス項目完了後に、テスト・検証ステップを明示する
4. エラーが発生したら次のステップへ進まず、停止して報告する

**Why:** usage レポートで「誤ったアプローチの初回選択」が10件の摩擦として記録された。また 1880% RTP の誤実装やレート制限によるセッション中断など、大きな変更を一度に行ったことで破壊的な結果になるケースが複数あった。

**How to apply:** 実装タスクが5ファイル以上またはロジックが複雑な場合は、実装前に「どのステップに分割するか」を提示して承認を得る。各ステップ後に「ここまで正常か確認してください」と明示する。
