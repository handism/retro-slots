---
name: 修正前に診断フェーズを必ず挟む
description: コンパイルエラーやバグ修正時は「診断→承認→修正」の順を守る
type: feedback
---

コンパイルエラー・バグ修正に着手する前に、必ず診断フェーズを完了させてから修正に入る。

**ルール:**
1. すべてのエラーをリストアップし、「根本原因」と「連鎖エラー」を分類する
2. 修正順序を提案してユーザーの承認を得る
3. 承認後に修正を実施する
4. 1件修正したら連鎖エラーが残っていないか確認してから「完了」と報告する

**Why:** usage レポートで最大の摩擦要因が「生成コードによる連鎖バグ」(14件) だった。SceneBuilder の5連続バグや RTP チューニングでの無限ループなど、診断なしに大きな変更を加えると新たなバグを生む悪循環が発生している。

**How to apply:** Unity C# コンパイルエラーや複雑なバグ修正の依頼が来たら、まず全エラーを列挙・分類して修正計画を示す。「ファイルを編集する前に承認を求める」パターンを使う。単純な1行修正は除く。
